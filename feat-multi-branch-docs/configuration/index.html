Configuration Reference

The configuration options below are derived from the Zephyr Kconfig definitions but represent the full set of tunable parameters for the Hubble Network SDK. The same options apply to all supported ports — Zephyr, FreeRTOS, ESP-IDF, bare-metal, and other RTOSes — though the mechanism for setting them varies by platform.

On Zephyr, set these in your application’s prj.conf file. On other platforms, define the equivalent CONFIG_ macros in your build system or project header.

EID generation mode

Choice — select one of:

CONFIG_HUBBLE_EID_TIME_BASED [default]

Unix Time-based EID generation

EID derived from Unix time. Requires time provisioning.

CONFIG_HUBBLE_EID_COUNTER_BASED

Counter-based EID generation

EID derived from internal counter. No Unix time required. Initial counter value provided via hubble_init().

CONFIG_HUBBLE_EID_ROTATION_PERIOD_SEC

EID rotation period in seconds

Type:

int

Default:

86400

Range:

86400 to 86400

EID rotation period. Currently only daily rotations are supported (86400)

CONFIG_HUBBLE_SAT_NETWORK

Hubble Satellite Network Library [PRE-PRODUCTION]

Type:

bool

Enable library for communication with Hubble network. This feature is in pre-production and is not yet ready for production deployments.

CONFIG_HUBBLE_SAT_NETWORK_SMALL

Use smaller code size for fly by calculation

Type:

bool

Depends on:

HUBBLE_SAT_NETWORK

Reduces code using polynomial approximation for trigonometric functions

CONFIG_HUBBLE_SAT_NETWORK_DEVICE_TDR

Device time drift retry rate in PPM

Type:

int

Default:

500

Depends on:

HUBBLE_SAT_NETWORK

Adds additional retries proportional to time since last time the device had time synced. It is represented in PPM (parts per million).

Hubble Sat Network protocol

Choice — select one of:

CONFIG_HUBBLE_SAT_NETWORK_PROTOCOL_V1 [default]

Use V1 protocol

First version of Sat protocol. Channel hopping within transmissions.

CONFIG_HUBBLE_BLE_NETWORK

Hubble BLE Network Library

Type:

bool

Enable library for communication with Hubble BLE network

Hubble Network key size

Choice — select one of:

CONFIG_HUBBLE_NETWORK_KEY_256

Use 256 bits key size

256 bits key size

CONFIG_HUBBLE_NETWORK_KEY_128

Use 128 bits key size

128 bits key size

CONFIG_HUBBLE_NETWORK_SECURITY_ENFORCE_NONCE_CHECK

Enforce check in nonce used in cryptography

Type:

bool

Default:

y

Depends on:

HUBBLE_BLE_NETWORK || HUBBLE_SAT_NETWORK

Ensure that nonce parameters are not re-used to encrypt different messages. Be aware that re-using the same nonce has severe security implications.

CONFIG_HUBBLE_NETWORK_SEQUENCE_NONCE_CUSTOM

Application defined implementation

Type:

bool

Depends on:

HUBBLE_BLE_NETWORK || HUBBLE_SAT_NETWORK

This option allows applications to override the default function that returns a sequence number to be used to encrypt messages.

CONFIG_HUBBLE_UPTIME_CUSTOM

Application defined uptime implementation

Type:

bool

Depends on:

HUBBLE_BLE_NETWORK || HUBBLE_SAT_NETWORK

This option allows applications to override the default function that returns the system uptime in milliseconds. This is primarily useful for unit testing where controlled time values are needed to verify time-dependent behavior. Implement hubble_uptime_get to override.

Hubble Network crypto provider

Choice — select one of:

CONFIG_HUBBLE_NETWORK_CRYPTO_MBEDTLS

Use MBEDTLS

Use MBEDTLS for cryptography.

CONFIG_HUBBLE_NETWORK_CRYPTO_PSA [default]

Use PSA Crypto

Use PSA Crypto for cryptography.

CONFIG_HUBBLE_NETWORK_CRYPTO_CUSTOM

Custom crypto implementation

Custom cryptography implementation. The target will implement all crypto interfaces.
